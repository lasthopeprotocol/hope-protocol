## ğŸ’€ What is $HOPE?

$HOPE is a **redistribution protocol** â€” not just a token. It autonomously identifies the holder experiencing the largest unrealized loss and compensates them using accumulated creator fees.

Every cycle, the protocol:
- Collects trading fees generated by the token
- Converts 100% of fees into $HOPE tokens
- **Airdrops 50%** to the wallet with the biggest loss
- **Burns 50%** permanently, reducing total supply

> *"The more it hurts, the more you earn."*

**Cost Basis Adjustment Formula:**

```

This prevents:

| Attack Vector | How It's Blocked |
|---------------|-----------------|

```

3. Loss shrinks â†’ other holders become top loser

> Rewards received do NOT affect cost basis â€” they only improve current value, gradually reducing the loss and rotating the top loser position to another holder.

Yes. The bot detects whether the token is on pump.fun bonding curve or has migrated to Raydium, and adjusts fee collection accordingly.
</details>

<details>
<summary><b>Is the source code the full protocol?</b></summary>

The repository contains the redistribution engine and frontend. Sensitive configuration (keys, RPC endpoints) is managed via environment variables and never committed.
</details>

<details>
<summary><b>What prevents the bot wallet from being drained?</b></summary>

The bot maintains a gas reserve and only uses accumulated creator fees for buybacks. It never risks the base wallet balance.
</details>
